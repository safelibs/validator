# Unpatched CVEs against Ubuntu 24.04 noble

This file tracks CVEs surfaced by safelibs port `relevant_cves.json`
files that were excluded from the regression testcase set because the
Ubuntu 24.04 noble baseline still exhibits the vulnerable behavior.
Each row records the library, CVE id, upstream evidence, and the
observation that prevented the testcase from passing on `original`.

| Library | CVE | Reason | Upstream evidence | Observation |
| --- | --- | --- | --- | --- |
| libjson | CVE-2013-6371 | not_reproducible_offline | json-c hash collision DoS fixed in 0.12 by salting string hashing with a random seed | No deterministic offline reproducer available — exercising the worst-case bucket distribution requires knowing the upstream hash function and crafting collisions that generalize across noble's libjson-c5 package; treated as covered structurally rather than as an executable regression. |
| libjansson | CVE-2013-6401 | not_reproducible_offline | Jansson 2.6 changed hash function and seeded hashes with random data | Hash-flooding regression cannot be encoded as a deterministic offline test against a randomized hasher; observable behavior on a single input is identical between vulnerable and patched builds. |
| libuv | CVE-2014-9748 | not_reproducible_offline | uv_rwlock_t fallback affected only Windows XP / Server 2003 | Linux harness does not exercise the Windows fallback rwlock path; the noble libuv1t64 build does not ship the affected code. |
| libuv | CVE-2015-0278 | not_reproducible_offline | Privilege drop in uv_spawn did not clear supplementary groups | A meaningful reproducer requires running as root, calling setgroups, and observing supplementary-group leakage in the spawned child; the validator harness does not have privileged container capabilities. |
| libuv | CVE-2024-22017 | not_reproducible_offline | setuid did not affect libuv's pre-initialized io_uring submissions | Demonstrating retained privileges across setuid requires an io_uring-capable kernel and a privileged-to-unprivileged transition that the noble container harness does not provide. |
| libxml | CVE-2008-4409 | not_reproducible_offline | Crafted predefined-entity definitions could exhaust memory | Public reproducers rely on internals of older libxml2 entity-table allocation; modern noble libxml2 has rewritten this path and there is no published canonical input that asserts patched behavior offline. |
| libxml | CVE-2012-0841 | not_reproducible_offline | Dictionary hash randomization mitigates collision DoS | Cannot construct deterministic adversarial keys against a randomized hash without knowledge of the per-process seed. |
| libxml | CVE-2013-0338 | not_reproducible_offline | Internal entity expansion linear-complexity DoS | The bug surfaces as wall-clock degradation on a vulnerable build; on a patched build the input parses identically to other billion-laughs variants, so a discriminating offline reproducer collapses into CVE-2003-1564 / CVE-2014-3660 rather than this CVE specifically. |
| libxml | CVE-2015-1819 | not_reproducible_offline | xmlReader streaming API failed constant-memory guarantee under entity expansion | Demonstrating the constant-memory regression requires measuring RSS over a long-running reader stream, which the matrix runner does not capture; the input itself is covered by the entity-expansion regressions. |
| libxml | CVE-2015-5312 | not_reproducible_offline | xmlStringLenDecodeEntities expansion-limit gap | Bypass converged with the general billion-laughs limits in upstream — same payloads now uniformly trigger the patched limit, so this CVE is not separable from CVE-2014-3660 in an offline test. |
| libxml | CVE-2015-8035 | not_reproducible_offline | XZ decompression error-handling loop did not terminate | Reproducer requires a crafted XZ-compressed XML whose error path drove the loop; the validator does not ship the malformed-xz fixture and constructing one offline that distinguishes vulnerable from patched behavior is not deterministic. |
| libxml | CVE-2016-4449 | not_reproducible_offline | Entity decoding outside validating mode fetched external content | Public reproducers depend on a network endpoint or a non-default loader callback; default noble configuration is covered by the XXE regressions for CVE-2013-0339 and CVE-2017-7375 and adds no separable signal for this CVE specifically. |
| libxml | CVE-2016-9318 | not_reproducible_offline | API design: lacked a same-document-only resolution policy | This is an API-surface concern: there is no input that distinguishes vulnerable from patched behavior at the xmllint command level; the patched API options exist but exercising them does not assert past-CVE state. |
| libxml | CVE-2017-18258 | not_reproducible_offline | LZMA decompression had no memory limit | Reproducer requires a crafted XZ-wrapped XML hitting xmlIO LZMA paths with attacker-driven memory allocation; no canonical fixture is shipped with the validator and constructing a discriminating one offline is not deterministic. |
| libxml | CVE-2018-9251 | not_reproducible_offline | LZMA memlimit-error infinite loop in libxml2 IO | Same shape as CVE-2017-18258 — distinguishing patched from vulnerable behavior requires an XZ fixture crafted to trigger the memlimit error path, which is not part of the validator fixture set. |
| libxml | CVE-2018-14567 | not_reproducible_offline | Second LZMA infinite-loop variant in the same code path | Variant of CVE-2018-9251 against the same crafted-XZ fixture; sharing the same offline reproducer constraint. |
| libxml | CVE-2021-3537 | not_reproducible_offline | Mixed-content parser-error propagation | Bug fired only when a specific schema-validation pipeline observed an error after a non-fatal parse; the public reproducer relies on internal reset paths that the patched libxml2 has refactored and no canonical input asserts the patched control flow offline. |
| libxml | CVE-2023-28484 | not_reproducible_offline | xmlSchemaFixupComplexType nullability bug | Public reproducer depends on a specific malformed XSD whose canonical form is not part of the validator fixture set; a discriminating offline input could not be constructed without risking false negatives on noble. |
| libxml | CVE-2025-27113 | not_reproducible_offline | xmlPatMatch nullability crash | The CVE record gives no concrete reproducer input; the patch added defensive null checks but no canonical malformed input is published, so an offline regression cannot distinguish vulnerable from patched builds. |
| libxml | CVE-2025-49795 | not_reproducible_offline | XPath null dereference on crafted expressions | Public reports describe the bug class but do not publish a deterministic input; XPath fuzz reproducers vary across libxml2 minor versions. |
| libxml | CVE-2026-0992 | not_reproducible_offline | Repeated nextCatalog traversal CPU exhaustion | The CPU-exhaustion regression requires measuring wall-clock degradation over many catalog hops; on a patched build the same input parses quickly, so the offline test collapses into CVE-2026-0990 rather than asserting this CVE separately. |
